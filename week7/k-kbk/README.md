# 로그 관리 및 시스템 모니터링

## 로그 관리

RHEL은 이벤트를 수집하고 분류하기 위해 `systemd-journald(journald)`와 `rsyslog`라는 두 가지 핵심 서비스를 사용합니다.

### systemd-journald

커널, 부팅 프로세스, 데몬의 표준 출력 및 syslog 이벤트를 수집하여 인덱싱된 구조적 바이너리 형태인 '시스템 저널'에 기록합니다. 기본적으로 `/run/log/journal`에 저장되어 재부팅 시 삭제되지만(휘발성), 설정을 통해 영구 저장할 수 있습니다.

### rsyslog

systemd-journald로부터 메시지를 읽어들여, 메시지의 기능(facility)과 우선순위(priority)에 따라 `/var/log` 디렉터리 내의 텍스트 파일로 분류하여 저장합니다.

### 주요 로그 파일

- `/var/log/messages`: 인증, 메일, 예약 작업 등을 제외한 대부분의 일반적인 syslog 메시지가 기록됩니다.
- `/var/log/secure`: 보안 및 인증(SSH 로그인 등)과 관련된 이벤트가 기록됩니다.
- `/var/log/maillog`: 메일 서버와 관련된 메시지가 저장됩니다.
- `/var/log/cron`: 예약된 작업(cron) 실행에 대한 메시지가 저장됩니다.
- `/var/log/boot.log`: 시스템 시작 시의 콘솔 메시지가 기록됩니다.

### journalctl을 활용한 시스템 로그 조회

바이너리 형태의 시스템 저널을 읽고 검색하기 위해서 `journalctl` 명령을 사용해야 합니다. 다양한 옵션을 조합하여 원하는 로그를 필터링할 수 있습니다.

- `journalctl` : 전체 로그를 표시합니다.
- `journalctl -n 5` : 최근 5개의 로그 항목만 표시합니다.
- `journalctl -f` : 로그 파일 끝에 새롭게 추가되는 내용을 실시간으로 출력합니다. (tail -f와 유사)
- `journalctl --since today` : 오늘 기록된 로그만 표시합니다.
- `journalctl --since "2026-04-01 00:00" --until "2026-04-05 10:00"` : 특정 기간 동안의 로그를 검색합니다.
- `journalctl -b` : 현재 시스템 부팅 이후의 로그 항목만 검색합니다.
- `journalctl -u sshd.service`: -u 옵션을 사용하여 특정 systemd 유닛(서비스)에 대한 메시지만 표시합니다.
- `journalctl -p err`: 에러(err) 등급 이상의 심각한 문제(err, crit, alert, emerg)만 필터링하여 표시합니다.
- `journalctl _PID=1234`: 특정 프로세스의 이벤트만 표시합니다.

## 시스템 모니터링

### 프로세스 상태 조회

- `ps aux`: 제어 터미널 유무와 상관없이 시스템의 모든 프로세스를 자세히 표시합니다. USER, PID, %CPU, %MEM, 프로세스 상태(STAT) 등을 확인할 수 있습니다.  
  \*프로세스 상태: R(실행 중), S(수면/대기), D(중단 불가능한 수면), T(중지됨), Z(좀비) 등
- `top`: 구성 가능한 간격으로 새로 고침되며, 시스템 부하, CPU 사용률(%Cpu), 메모리 및 스왑 사용량, 그리고 리소스를 가장 많이 사용하는 프로세스 목록을 실시간으로 표시합니다. `M`키를 누르면 메모리 사용량 순으로, `P`키를 누르면 CPU 사용량 순으로 정렬할 수 있습니다.

### 디스크 상태 조회

- `df -h`: 마운트된 전체 파일 시스템의 총 디스크 공간, 사용량, 남은 공간을 사람이 읽기 쉬운 단위(MB, GB)로 표시합니다.
- `du -h /path`: 특정 디렉터리 트리 내의 파일들이 차지하는 디스크 공간을 재귀적으로 계산하여 표시합니다.

### 기타 시스템 점검 도구

- `uptime`: 현재 시간, 가동 시간, 로그인한 사용자 수와 함께 최근 1분, 5분, 15분 동안의 부하 평균을 출력합니다.
- `lscpu`: 시스템의 아키텍처, 소켓 수, 코어 수, 스레드 수 등 CPU의 물리적/논리적 구성을 파악할 수 있습니다.
- `free -h`: 시스템의 물리적 메모리(RAM)와 스왑(Swap) 공간의 사용 및 여유 상태를 MB, GB 등의 단위로 출력합니다.  
  \*`-h`: human-readable

### 작업 제어 (Job Control)

- 명령어 뒤에 `&`를 붙여 Background로 실행하고 , `jobs` 명령어로 작업 목록을 확인합니다. `Ctrl + Z`로 Foreground 작업을 일시 중지 시킬 수 있으며, `fg`와 `bg` 명령을 통해 Foreground/Background로 상태를 전환할 수 있습니다.
