# CPU·메모리·디스크·프로세스 사용량 점검

## 자원 사용량 데이터 흐름 구조
```
[ Kernel (자원 관리자) ]
           |
           +------------------------------------------+
           | (실시간 데이터 노출)                        | (이벤트 발생 시)
           v                                          v
+------------------------+                  +---------------------------+
|  /proc & /sys (가상FS)  |                  |    systemd-journald       |
| (CPU, Mem, Disk 정보)   |                  | (OOM Killer, Disk Full)   |
+------------------------+                  +---------------------------+
           |                                          |
     +-----+-----+                                    | (바이너리 로그)
     |           |                                    v
 [Top / Free] [iostat]                       [ journalctl -p err ]
 [ vmstat  ]  [ df -h ]                      (임계치 초과 기록 확인)
```

## 각 자원별 점검 원리

### CPU와 프로세스 (Compute)
- 원리 : 커널은 각 프로세스가 점유한 타임 슬롯을 `/proc/[PID]/stat`에 기록  
    -> `top`이나 `ps`는 해당 파일을 읽어 백분율로 계산
- 결과 : 어떤 프로세스가 연산 자원을 독점하여 시스템 응답성을 ㄸ러어뜨리는지 식별
- 확인 지표
  - Load Average : CPU 코어 수 대비 대기 중인 프로세스 수 확인 (1.0 이면 1코어 100% 사용 중)
  - iowait (`%wa`) : CPU는 노는데 응답이 느려 처리가 되지 않는 병목 현상 식별
- 명령어
  - `top` (`htop`) : 전체적인 PCU 사용륭 및 프로세스 순위 화긴
  - `uptime` : 시스테 부하 지수 확인
  - `vmstat 1` : CPU의 사용자(`us`), 시스템(`sy`), 대기(`wa`) 비주 추이 관찰 

### 메모리 (Memory)
- 원리 : `/proc/meminfo`를 통해 사용 가능 메모리와 캐시/버퍼 상태 확인
- 로그의 역할 : 메모리가 고괄되면 커널의 OOM Killer가 작동하여 기록이 journald에 바이너리 형태로 기록
- 결과 : 메모리 누수 발생 여부와 강제 종료 프로세스 확인
- 확인 지표
  - Swap In/Out : 물리 메모리 부족으로 디스크를 메모리처럼 쓰면서 시스템이 느려지는 현상 감지
- 명령어
  - `free -h` : 사요 가능한 실제 메모리량 확인
  - `sar -B 1` : 페이지 폴트 및 스와핑 발생 여부 실시간 모니터링
  - `journalctl -k | grep -i oom` : 메모리 부족으로 강제 종료된 프로세스 추적

### 디스크 (Storage)
- 원리 : 파일 시스템의 슈퍼블록 정보를 조회하여 사용량 파악
- 결과 : 용량 부족으로 인한 서비스 중단 가능성을 사전에 파악
- 확인 지표
  - Inode 사용량 : 용량은 남았는데 파일 개수가 꽉 차서 파이 생성이 되지 않는 장애 방지
- 명령어
  - `df -h / df -i` : 디스크 용량 및 Inode 사용량 확인
  - `iostat -xz 1` : 디스크별 응답 시간 및 부하 분석 

### 네트워크 (Network)
- 원리 : 커널이 `/proc/net/dev`에 인터페이스벼 패킷 수와 에러 수 기록
- 결과 : 서비스 응답 속도가 느릴 때 이거이 서버 연산 문제인지 넽워크 통로 문제인지 판별
- 확인 지표
  - RX/TX Errors : 하드웨어 문제나 케이블 불량으로 패킷이 깨지는지 확인
  - Dropped Packets : 커널 처리 용량을 넘어서서 들어오는 패킷을 버리고 있느지 (부하 과다) 확인 
- 명령어
  - `ip -s link` : 인터페이스별 에러/드랍 패킷 통계 확인
  - `sar -n DEV 1` : 실시간 초당 전송량 및 패킷 수 확인
  - `ss -tulpn` : 현재 열려 있는 포트와 통신 중인 프로세스 연결 상태 확인
