# journalctl을 활용한 시스템 로그 조회

## journalctl 조회 고정


### 내부 동작 원리 
```
[ 사용자 명령 ]
     journalctl -u sshd
            |
            v
+-----------------------------+
|      journalctl 엔진        | <--- 1. 명령 해석 (Filter: UNIT=sshd)
+-----------------------------+
            |
    [ Index Search ] <----------- 2. 바이너리 파일 내 인덱스 테이블 탐색
            |                    (텍스트 전체 검색보다 훨씬 빠름)
            v
+-----------------------------+
| /run/log/journal/데이터파일  | <--- 3. 조건에 맞는 바이너리 블록 추출
+-----------------------------+
            |
    [ 포맷팅 변환 ] <------------- 4. 바이너리 -> 사람이 읽는 텍스트로 변환
            |
            v
    [ 터미널 출력 ]
```
### 구조적 특징
``` bash
$ journalctl -u sshd -n ` -o json | jq
```
- 단순한 메시지 이외의 값들 확인 가능
  - `_PID` : 로그를 남긴 프로세스 ID
  - `_UID` : 로그를 실행한 사용자 ID
  - `_HOSTNAME` : 발생한 호스트 이름
  - `_SYSTEMD_UNIT` : 관련 시스템 유닛
  - `_COMM` : 실행 파일 이름


## 주요 조회 방법

### 시간 기반 조회
- 로그의 타임 스탬프 인덱스를 사용하여 특정 시점의 로그 확인

### 서비스 기반 조회
- 시스템 서비스 단위로 로그를 묶어 확인 

### 중요도 기반 조회
- 로그 레벨을 지정하여 긴급한 상황만 골라 조회
