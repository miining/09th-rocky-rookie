# RHEL 10 기반 Bash 셸 스크립트 완벽 이해

셸의 기본 개념부터 자동화 스크립트 구조까지 — RHEL 10 스터디를 위한 Bash 셸 스크립트 학습 자료입니다.

---

## 1. 셸(Shell)의 근본 개념과 동작 원리

리눅스를 이해하기 위해 가장 먼저 알아야 할 것은 '셸(Shell)'이 정확히 무엇인지 그 정체를 파악하는 것입니다.

### 셸(Shell)의 정의

셸은 사용자와 운영 체제의 핵심인 **커널(Kernel)** 사이에서 인터페이스 역할을 하는 프로그램입니다. 사용자가 키보드를 통해 명령어를 입력하면, 셸은 이 텍스트를 읽고 해석하여 커널이 이해할 수 있는 형태로 전달하고, 그 실행 결과를 다시 화면(터미널)에 출력해 줍니다.

### 명령어 해석과 `$PATH`의 역할

터미널에 명령어를 입력하고 엔터를 누르면, 셸은 먼저 해당 명령어가 셸 자체에 내장된 **빌트인(Built-in) 명령어**인지 확인합니다. 만약 빌트인 명령어가 아니라면, 셸은 환경 변수인 `$PATH`에 정의된 디렉터리 목록을 왼쪽부터 오른쪽으로 순차적으로 검색하며 해당 이름의 실행 파일이 있는지 찾습니다. 실행 파일을 찾으면 이를 실행하고, 찾지 못하면 `command not found`라는 에러 메시지를 출력합니다.

### Bash(Bourne-Again SHell)와 RHEL 10

Bash는 유닉스/리눅스 환경에서 가장 널리 사용되는 표준 셸 프로그램입니다. 수많은 명령어들을 하나씩 직접 치는 대신, 텍스트 파일에 순서대로 적어두고 셸이 이를 한 줄씩 읽어 실행하게 만드는 것이 바로 **셸 스크립트(Shell Script)**입니다. RHEL 10은 성능이 크게 향상된 **Bash 5.2** 버전(`bash-5.2.26`)을 기본적으로 탑재하고 있어 구문 분석 및 메모리 관리가 더욱 안정적입니다.

## 2. 스크립트의 해부학: 시뱅(Shebang)과 권한

단순한 텍스트 파일이 독립적인 스크립트 프로그램으로 동작하기 위해서는 반드시 필요한 두 가지 요소가 있습니다.

### 시뱅 (Shebang, `#!`)

모든 Bash 스크립트의 맨 첫 줄은 반드시 `#!` 문자로 시작해야 합니다. 이 기호 조합을 **시뱅(Shebang)**이라고 부르며, 운영 체제에게 이 텍스트 파일을 어떤 인터프리터(해석기)를 사용해 실행할 것인지 그 절대 경로를 알려주는 역할을 합니다.

| 시뱅 형태 | 설명 |
|---|---|
| `#!/bin/bash` | RHEL 시스템에서 일반적으로 사용하는 형태 |
| `#!/usr/bin/env bash` | 환경 변수에서 동적으로 Bash를 찾아 여러 시스템 간 호환성을 높이는 형태 |

만약 이 줄이 누락되면 시스템은 기본 셸로 스크립트를 해석하려 시도하다가 예상치 못한 오류를 발생시킬 수 있습니다.

### 실행 권한 부여 (Permissions)

리눅스의 파일 시스템은 보안을 위해 새로 생성된 텍스트 파일에 실행 권한을 부여하지 않습니다. 따라서 스크립트 파일을 실행하려면 `chmod` 명령어를 사용해 명시적으로 권한을 변경해야 합니다.

```bash
chmod +x script.sh      # 모든 사용자에게 실행 권한 부여
chmod u+x script.sh     # 소유자에게만 실행 권한 부여
```

권한 부여 후 `./script.sh` 형태로 스크립트를 독립된 프로그램처럼 실행할 수 있습니다.

## 3. 변수와 동적 데이터 관리 (Variables & Parameters)

Bash 스크립트는 명시적인 데이터 타입(정수, 문자열 등)을 선언할 필요가 없는(Untyped) 유연한 언어입니다.

### 변수의 선언과 호출

변수를 선언할 때는 **공백 없이** `myvar="value"` 형태로 작성해야 하며, 저장된 값을 불러올 때는 이름 앞에 `$` 기호를 붙여 `$myvar` 형태로 호출합니다.

```bash
myvar="hello"
echo $myvar     # hello 출력
```

### 변수의 범위 (Scope)

| 종류 | 선언 방법 | 설명 |
|---|---|---|
| 전역 변수 | `myvar="value"` | 스크립트 내 어디서나 접근 가능 (기본값) |
| 지역 변수 | `local myvar="value"` | 함수 내부에서만 존재, 의도치 않은 데이터 덮어쓰기 방지 |
| 환경 변수 | `export myvar="value"` | 자식 프로세스에서도 사용 가능하도록 승격 |

### 위치 매개변수 (Positional Parameters)

스크립트를 실행할 때 외부에서 전달하는 입력값들입니다. 스크립트의 유연성과 재사용성을 높이는 핵심 요소입니다.

| 매개변수 | 설명 |
|---|---|
| `$0` | 실행된 스크립트 파일의 이름 |
| `$1` ~ `$9` | 첫 번째부터 아홉 번째까지의 입력 인수 (10 이상은 `${10}` 사용) |
| `$#` | 스크립트에 전달된 총 인수의 개수 |
| `$@` | 전달된 모든 인수. `"$@"`로 사용하면 인수 내 공백을 보존하여 단어 분리 오류를 방지 |
| `$?` | 가장 최근에 실행된 명령어의 종료 상태 (성공: `0`, 실패: `1`~`255`) |

> **팁:** 엔터프라이즈 환경에서는 인수에 포함된 공백 보존을 위해 반드시 `"$@"`(따옴표 포함)를 사용합니다.

## 4. 데이터 스트림과 리다이렉션 (Data Streams & Redirection)

Bash에서 실행되는 모든 프로세스는 커널과 통신하기 위해 3개의 표준 **파일 디스크립터(File Descriptor)** 스트림을 가집니다.

### 표준 스트림

| 스트림 | FD 번호 | 설명 |
|---|:---:|---|
| 표준 입력 (stdin) | `0` | 키보드 등을 통해 프로그램으로 들어가는 데이터 |
| 표준 출력 (stdout) | `1` | 명령어가 정상적으로 실행되어 터미널 화면에 나타나는 결과 |
| 표준 에러 (stderr) | `2` | 명령어 실행 중 발생한 오류나 진단 메시지 |

### 리다이렉션 (출력 방향 바꾸기)

| 연산자 | 동작 |
|---|---|
| `>` | 표준 출력을 파일로 보냄. 파일이 이미 존재하면 **덮어씀** |
| `>>` | 표준 출력을 파일 **끝에 추가(Append)** |
| `2>` | 에러 메시지만 특정 파일로 저장 |
| `&>` 또는 `> file 2>&1` | 정상 출력과 에러 메시지를 모두 하나의 파일로 병합하여 저장 |

> **활용:** 자동화된 백업 및 로그 스크립트에서 `&>` 연산자가 매우 자주 사용됩니다.

### 파이프 (`|`)

앞선 명령어의 **표준 출력**을 뒤에 오는 명령어의 **표준 입력**으로 실시간으로 전달합니다. 여러 명령어를 조립하여 강력한 데이터 필터링 파이프라인을 구축할 수 있습니다.

```bash
cat file.txt | grep "error"
```

## 5. 제어 구조 (Control Structures: 조건문과 반복문)

단순히 순서대로 실행되는 것을 넘어, 판단을 내리고 작업을 반복하는 '지능형' 스크립트를 만드는 구조입니다.

### 조건 분기 (`if`, `elif`, `else`)

조건의 참/거짓에 따라 실행 흐름을 나눕니다. RHEL 10 최신 환경에서는 단일 대괄호 `[ ]`보다 **확장된 이중 대괄호 `[[ ... ]]`** 사용이 강력히 권장됩니다.

- 내부에서 `&&`(AND), `||`(OR) 연산자를 안전하게 사용 가능
- `=~`를 통한 정규 표현식 매칭 지원

```bash
if [[ -f "$file" && "$count" -eq 0 ]]; then
    echo "파일이 존재하고 카운트가 0입니다."
elif [[ -d "$file" ]]; then
    echo "디렉터리입니다."
else
    echo "해당 없음"
fi
```

**주요 조건 플래그:**

| 플래그 | 설명 |
|---|---|
| `-f` | 파일 존재 확인 |
| `-d` | 디렉터리 존재 확인 |
| `-eq` | 숫자 일치 확인 |
| `-ne` | 숫자 불일치 확인 |
| `-z` | 문자열이 비어 있는지 확인 |

### 다중 패턴 매칭 (`case`)

하나의 변수를 여러 가지 문자열 패턴과 비교할 때 사용합니다. 복잡하게 얽힌 `if-elif` 문보다 가독성이 훨씬 뛰어나 시스템 관리용 스크립트의 옵션 파싱 등에 자주 쓰입니다.

```bash
case "$1" in
    start)  echo "서비스 시작" ;;
    stop)   echo "서비스 중지" ;;
    *)      echo "사용법: $0 {start|stop}" ;;
esac
```

### 반복문 (`for`, `while`, `until`)

| 반복문 | 용도 |
|---|---|
| `for` | 파일 목록, 배열, `{1..5}`와 같은 숫자 범위를 순차적으로 순회 |
| `while` | 주어진 조건이 **참(True)인 동안** 반복. 로그 파일 실시간 읽기 등에 활용 |
| `until` | `while`과 반대로 조건이 **참이 될 때까지**(거짓인 동안) 반복 |

```bash
# for 루프 예시
for i in {1..5}; do
    echo "반복 $i"
done

# while 루프 예시
while [[ "$count" -lt 10 ]]; do
    ((count++))
done
```

**루프 제어:**
- `break`: 루프를 즉시 강제 종료
- `continue`: 현재 주기의 남은 코드를 건너뛰고 다음 주기로 이동

## 6. 엔터프라이즈 환경을 위한 방어적 스크립팅 (Strict Mode)

RHEL 서버와 같은 프로덕션(운영) 환경에서 자동화 스크립트가 오작동하여 시스템을 망가뜨리는 것을 방지하기 위해, 스크립트 상단(시뱅 바로 아래)에 반드시 추가해야 하는 설정입니다.

```bash
#!/bin/bash
set -euo pipefail
```

### 각 옵션의 역할

| 옵션 | 역할 |
|---|---|
| `-e` | 스크립트 내 어떤 명령어가 에러(0이 아닌 종료 상태)를 반환하면 **즉시 실행 중단**. 실패 상태로 다음 명령이 실행되어 발생할 수 있는 연쇄 사고를 방지 |
| `-u` | 초기화되지 않은(정의되지 않은) 변수를 사용하면 **에러로 간주하고 중단**. 변수명 오타로 인한 사고 방지 |
| `-o pipefail` | 파이프(`\|`)로 연결된 명령 중 **단 하나라도 실패하면 파이프라인 전체를 실패로 처리** |

### Strict Mode 적용 예시

```bash
#!/bin/bash
set -euo pipefail

BACKUP_DIR="/backup"
SOURCE_DIR="/data"

# -u: SOURCE_DIR이 정의되지 않았다면 여기서 즉시 중단
tar -czf "${BACKUP_DIR}/data.tar.gz" "${SOURCE_DIR}" | tee backup.log
# -o pipefail: tar 또는 tee 중 하나라도 실패하면 스크립트 중단
```

> **핵심 원칙:** 프로덕션 환경의 모든 자동화 스크립트에는 `set -euo pipefail`을 반드시 포함시킵니다.
