## 로키 리눅스(Rocky Linux)의 등장 배경

2020년 12월, 레드햇(Red Hat)은 기존의 CentOS Linux 지원을 단계적으로 종료하고, 향후 개발 역량을 CentOS Stream에 집중하겠다고 공식 발표했습니다.

기존의 CentOS Linux는 상용 운영체제인 RHEL(Red Hat Enterprise Linux)의 소스코드를 100% 가져와 상표권 등의 법적 제약 요소만 제거하고 재빌드한 다운스트림(Downstream) 운영체제였습니다. 엔터프라이즈 수준의 안정성과 RHEL과의 완벽한 호환성을 무료로 제공했기 때문에, 전 세계 서버 시장에서 매우 높은 점유율을 차지하고 있었습니다.

하지만 레드햇의 이러한 정책 변경으로 인해, 안정적인 무료 RHEL 클론 운영체제를 운영하던 수많은 기업과 사용자들은 대안을 찾아야 하는 상황에 직면했습니다.

이러한 배경 속에서 기존 CentOS 프로젝트의 공동 창립자인 그레고리 커처(Gregory Kurtzer)가 기존 CentOS의 초기 목표와 철학을 이어가는 새로운 오픈소스 프로젝트를 창설했습니다. 고인이 된 또 다른 공동 창립자 로키 맥고(Rocky McGaugh)의 이름을 기려 로키 리눅스(Rocky Linux)라는 이름으로 배포되었으며, RHEL과 버그 단위까지 완벽하게 호환되는 엔터프라이즈급 운영체제를 지향하고 있습니다.

> Linux의 정확한 명칭은 [GNU/Linux](https://www.gnu.org/gnu/why-gnu-linux.ko.html)입니다.

## RHEL 생태계의 구조

![image.png](https://github.com/user-attachments/assets/ca4f210f-3cb5-4d8c-a121-68488145b85a)

로키 리눅스의 위치를 정확히 이해하기 위해서는 소스코드의 흐름에 따른 RHEL 생태계의 구조를 파악해야 합니다. 이 생태계는 크게 네 가지 단계로 분류됩니다.

- Fedora (업스트림): 최신 기술과 혁신적인 기능을 가장 먼저 도입하고 테스트하는 커뮤니티 주도의 운영체제입니다. 업데이트 주기가 짧고 최신 패키지를 제공하지만, 프로덕션 서버용으로는 실험적인 성격이 강합니다.
- CentOS Stream (미드스트림): Fedora에서 1차 검증된 기술이 RHEL에 정식으로 도입되기 직전에 거치는 롤링 릴리스(Rolling Release) 형태의 운영체제입니다. 향후 출시될 RHEL 마이너 업데이트의 미리보기 역할을 수행합니다.
- RHEL (상용 버전): 안정성과 보안을 최우선으로 검증하여 출시되는 레드햇의 공식 상용 운영체제입니다. 유료 구독을 통해 기술 지원과 장기적인 유지보수를 제공받습니다.
- Rocky Linux (다운스트림): RHEL 정식 버전이 출시될 때 공개되는 소스코드를 바탕으로 새롭게 빌드된 무료 운영체제입니다. 과거 CentOS Linux가 담당하던 역할을 그대로 물려받아, 유료 구독 없이도 RHEL과 동일한 환경을 구축할 수 있게 해줍니다.

> Linux는 Unix 계열의, Unix와 비슷한(Unix-like) 운영체제입니다.  
> Unix 기반의 운영체제에는 macOS, AIX 등이 있습니다.  
> Windows는 Unix와 무관한 독자적인 운영체제입니다.

> Debian을 기반으로 한 배포판에는 Ubuntu, Kali Linux, Raspberry Pi OS 등이 있습니다.

## 리눅스 구성 요소

- 하드웨어(Hardware)
  - 시스템을 구동하는 물리적 기반 장치입니다. 중앙처리장치(CPU), 메모리, 하드 디스크, 네트워크 카드 등의 부품으로 구성되며, 시스템의 처리 능력과 한계를 결정합니다.
- 커널(Kernel)
  - 운영체제의 심장 역할을 하는 핵심 소프트웨어입니다. 하드웨어 자원을 직접 통제하고 응용 프로그램에 공평하게 분배합니다. 메모리 할당, 프로세스 스케줄링, 파일 시스템 관리, 네트워크 데이터 전송 등 시스템 운영에 필요한 모든 기초적인 작업을 담당합니다.
- 시스템 라이브러리(System Library)
  - 응용 프로그램이 커널의 기능을 사용할 수 있도록 연결해 주는 표준 규격 함수들의 집합입니다. 소프트웨어 개발자가 복잡한 하드웨어 제어 방식을 직접 코딩하지 않더라도, 라이브러리를 통해 커널에 작업 요청(시스템 콜)을 보낼 수 있게 해줍니다.
- 셸(Shell)
  - 사용자와 커널 사이의 소통을 전담하는 명령어 해석기입니다. 사용자가 터미널 환경에서 텍스트 명령어를 입력하면, 셸이 해당 명령어의 의미를 분석하여 커널이 이해할 수 있는 형태로 전달합니다. 작업이 완료되면 커널로부터 결과를 받아 사용자 화면에 출력합니다. 대표적인 셸 프로그램으로는 bash, zsh, csh 등이 있습니다.
- 응용 프로그램 및 시스템 유틸리티
  - 사용자가 특정한 목적을 달성하기 위해 실행하는 최상위 계층의 소프트웨어들입니다. 파일 복사나 디렉터리 생성 같은 기본적인 시스템 명령(cp, mkdir 등)부터 웹 서버, 데이터베이스 관리 시스템, 텍스트 편집기까지 사용자가 설치한 모든 프로그램이 여기에 해당합니다.
